# Modular Wagtail Page Loading System

This directory uses Python factories to generate fake CMS data for local development,
review apps, and visual regression testing.

## Overview

- Fake data is generated programmatically using Python factories.
- Block and snippet factories are reusable across pages.
- Data generation is reproducible with a fixed seed.

## How to Generate Fake Pages

Run:

    python manage.py load_redesign_data --force

This will:
- Generate the homepage and all related content using factories.
- Assign the homepage as the default site root.
- Generate navigation, footer, profiles, gallery, and other sections.

## Adding a New Fake Page

1. Create a new page factory (e.g. `core/factories/my_page.py`) using block factories from `blocks/factories.py`.
2. Add a `generate(seed=..., slug=...)` function.
3. Register your generator in `load_redesign_data.py`.

## Block Factories

- Use block factories from `blocks/factories.py` to compose StreamField content.

## Visual Regression (Percy)

- Pages generated by factories are ready for Percy or other visual regression tools.

## Conventions

- Do not add or edit JSON manifests for fake data.
- Keep block logic in `blocks/factories.py`.
- Keep snippet logic in `snippets/factories.py`.

## Troubleshooting

- If a block serialization error appears, check that all block fields receive primitive values (e.g. use `newsletter_signup.id`, not the object).
- If an image is missing, verify it exists in the expected images directory.
